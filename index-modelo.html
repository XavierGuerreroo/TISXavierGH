¿Qué es el modelo relacional?

El modelo relacional describe cómo las tablas de la base de datos de la Cafetería Monky están interconectadas entre los subsistemas de Facturación y Membresía.

Permite entender de forma visual cómo se gestionan los clientes, sus membresías y las facturas con sus detalles, pagos y cancelaciones, mostrando las llaves primarias y foráneas que vinculan la información.

Tablas principales

Cliente

Contiene la información básica de cada cliente: nombre, correo y teléfono.

Membresía

Registra el tipo de membresía, fechas de inicio y vencimiento, estado, historial de pagos y beneficios asociados.

Factura

Guarda los registros de ventas: folio, fecha, importe, método de pago, timbrado fiscal y relación con el cliente.

Detalle_Factura

Desglosa los productos vendidos en cada factura, vinculando la Carta con la cantidad, precio unitario y subtotal.

Pago

Permite registrar múltiples métodos de pago para una misma factura, como efectivo, tarjeta o vales.

Cancelación

Almacena la información de facturas canceladas: motivo, fecha y estado.

Carta

Catálogo de productos de la cafetería: cada ítem tiene un nombre, descripción y precio. Se conecta con Detalle_Factura para registrar las ventas.
